Security Layer는 5개의 패키지로 구성된 Bun 모노레포입니다. 각 패키지는 단일 책임과 명확한 경계를 가지고 있습니다.

## 아키텍처

```
┌──────────────────────────────────────────────────────┐
│                   securitylayer (CLI)                 │
│          사용자 대상 명령어, 설정, hooks               │
├──────────────────────────────────────────────────────┤
│                                                      │
│  ┌──────────────┐  ┌──────────────┐  ┌────────────┐ │
│  │    proxy      │  │     sdk      │  │   rules    │ │
│  │  WebSocket    │  │  프로세스 내  │  │   YAML     │ │
│  │  게이트웨이    │  │  검사        │  │  규칙 팩    │ │
│  └──────┬───────┘  └──────┬───────┘  └─────┬──────┘ │
│         │                 │                │         │
│  ┌──────┴───────┐         │                │         │
│  │   adapters    │         │                │         │
│  │  에이전트 프레 │         │                │         │
│  │  임 파싱      │         │                │         │
│  └──────┬───────┘         │                │         │
│         │                 │                │         │
│  ┌──────┴─────────────────┴────────────────┴───────┐ │
│  │                    core                          │ │
│  │  Capability, taint, 규칙 엔진, 파이프라인,       │ │
│  │  정규화, 송신 스캔, LLM 심판                      │ │
│  └──────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

## 패키지 개요

| 패키지 | npm | 설명 |
|---|---|---|
| [`@securitylayerai/core`](/docs/packages/core) | `@securitylayerai/core` | 보안 엔진 — capability, taint, 규칙, 파이프라인, 정규화 |
| [`@securitylayerai/rules`](/docs/packages/rules) | `@securitylayerai/rules` | 기본 규칙 및 capability 템플릿 (YAML) |
| [`@securitylayerai/adapters`](/docs/packages/adapters) | `@securitylayerai/adapters` | 에이전트 프로토콜 어댑터 (OpenClaw, generic) |
| [`@securitylayerai/proxy`](/docs/packages/proxy) | `@securitylayerai/proxy` | 클라이언트와 에이전트 게이트웨이 간 WebSocket 보안 프록시 |
| [`@securitylayerai/sdk`](/docs/sdk) | `@securitylayerai/sdk` | 프로세스 내 보안 검사를 위한 TypeScript SDK |
| `securitylayer` | `securitylayer` | CLI — 사용자 대상 명령어, 설정, hooks |

## 의존성 그래프

```
securitylayer (CLI)
├── @securitylayerai/core
├── @securitylayerai/rules
└── @securitylayerai/proxy
    └── @securitylayerai/adapters

@securitylayerai/sdk
└── @securitylayerai/core

@securitylayerai/rules
└── (독립형 — YAML만, 코드 의존성 없음)
```

주요 제약 사항:
- **core**는 내부 의존성이 없습니다 — 기반 패키지입니다
- **rules**는 데이터 전용입니다 — 얇은 로더가 포함된 YAML 파일이며 런타임에 core에 대한 의존성이 없습니다
- **adapters**는 독립형입니다 — 에이전트 프로토콜의 인터페이스와 구현을 정의합니다
- **proxy**는 프레임 파싱을 위해 adapters에 의존합니다
- **sdk**는 보안 파이프라인을 위해 core에 의존합니다

## 개발

```bash
# 모든 의존성 설치
bun install

# 모든 테스트 실행
bun run test

# 특정 패키지의 테스트 실행
bun run test --filter=@securitylayerai/core

# 전체 타입 검사
bun run typecheck
```
