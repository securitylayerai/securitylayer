`@securitylayerai/sdk`는 `@securitylayerai/core`를 깔끔하고 고수준의 API로 래핑하여 Security Layer를 애플리케이션 코드에 직접 통합할 수 있게 합니다. CLI와 프록시가 시스템 경계에서 동작하는 반면, SDK는 전체 컨텍스트와 함께 프로세스 내 보안을 제공합니다.

## 주요 내보내기

### createSecurityLayer()

메인 진입점입니다. 전체 security pipeline을 통해 동작을 평가하는 `SecurityLayer` 인스턴스를 생성합니다.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer({
  configDir: "~/.securitylayer",
  sessionId: "my-session",
});

const result = await sl.check("exec", { command: "ls -la" });
console.log(result.decision); // "ALLOW" | "DENY" | "REQUIRE_APPROVAL"

sl.destroy();
```

테스트를 위한 의존성 주입을 지원합니다 — 미리 로드된 `config`이나 `eventBus`를 전달하여 파일 I/O를 생략할 수 있습니다.

### withSecurityLayer()

모든 함수에 보안 검사를 투명하게 추가하는 미들웨어 래퍼입니다.

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

const safeExec = withSecurityLayer(sl, exec, "exec");
const safeWrite = withSecurityLayer(sl, writeFile, "file.write");

// 원본과 동일한 API — 보안 검사가 자동으로 수행됩니다
await safeExec("echo hello");
await safeWrite("/tmp/out.txt", data);
```

DENY 시 `SecurityLayerError`를 던집니다. REQUIRE_APPROVAL 시 승인을 대기합니다.

### 에러 클래스

각 실패 모드에 대한 타입이 지정된 에러:

| 에러 | 설명 |
|---|---|
| `SecurityLayerError` | 기본 에러 클래스; 미들웨어에서 DENY 시에도 던져짐 |
| `ConfigError` | 설정 파일을 로드하거나 파싱할 수 없음 |
| `InitializationError` | Pipeline 초기화에 실패 |
| `CheckError` | 동작 평가가 예기치 않게 실패 |
| `ApprovalTimeoutError` | 승인 요청이 시간 초과됨 |

## SecurityLayer 인터페이스

`createSecurityLayer()`가 반환하는 객체는 다음을 제공합니다:

| 메서드 | 설명 |
|---|---|
| `check(tool, params, context?)` | Security pipeline을 통해 동작을 평가합니다 |
| `waitForApproval(id, options?)` | 승인이 허용되거나 시간 초과될 때까지 대기합니다 |
| `ingestContent(content, metadata)` | 외부 콘텐츠로 taint tracking을 업데이트합니다 |
| `scanEgress(content)` | 발신 콘텐츠에서 유출된 비밀을 스캔합니다 |
| `getSessionTaint()` | 현재 세션의 유효 taint level을 가져옵니다 |
| `on(type, handler)` | 보안 이벤트를 구독합니다 |
| `destroy()` | 리소스와 이벤트 핸들러를 정리합니다 |

## 아키텍처

SDK는 `@securitylayerai/core`에만 의존하며 내부적으로 security pipeline을 구축합니다:

```
@securitylayerai/sdk
├── createSecurityLayer() ─→ loads config, builds pipeline deps
│   ├── CapabilityStore ─→ from core's buildCapabilityStore()
│   ├── TaintTracker ─→ from core's createTaintTracker()
│   ├── LLM Judge ─→ from core's createDefaultLLMJudge() or createNoOpJudge()
│   ├── ApprovalManager ─→ from core's createApprovalManager()
│   └── EventBus ─→ from core's createEventBus()
├── withSecurityLayer() ─→ middleware wrapper over check()
└── Error classes ─→ SecurityLayerError, ConfigError, etc.
```
