## AbhГӨngigkeitsgraph

```
securitylayer (CLI)
в”ңв”Җв”Җ @securitylayerai/core
в”ңв”Җв”Җ @securitylayerai/rules
в””в”Җв”Җ @securitylayerai/proxy
    в””в”Җв”Җ @securitylayerai/adapters

@securitylayerai/sdk
в””в”Җв”Җ @securitylayerai/core

@securitylayerai/rules
в””в”Җв”Җ (eigenstГӨndig вҖ” nur YAML, keine Code-AbhГӨngigkeiten)
```

Zentrale EinschrГӨnkungen:
- **core** hat null interne AbhГӨngigkeiten вҖ” es ist das Fundament
- **rules** ist rein datenbasiert вҖ” YAML-Dateien mit einem schlanken Loader, keine LaufzeitabhГӨngigkeit von core
- **adapters** ist eigenstГӨndig вҖ” definiert das Interface und Implementierungen fГјr Agentenprotokolle
- **proxy** hГӨngt von adapters fГјr das Frame-Parsing ab
- **sdk** hГӨngt von core fГјr die Sicherheitspipeline ab
