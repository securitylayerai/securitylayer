`@securitylayerai/rules` es un paquete de datos -- definiciones de rules en YAML y capability templates con un loader ligero en TypeScript. No tiene dependencia de runtime con core. El CLI y otros paquetes consumen el YAML en tiempo de ejecucin usando `parseRulesYaml()` de core.

## Contenido

### Baseline rules (`rules/baseline.yaml`)

14 rules deterministas que complementan las rules integradas de core. Cubren patrones que las rules integradas no contemplan:

| Categora | ID de rule | Decisin | Qu detecta |
|---|---|---|---|
| Destructivo | `destructive-chmod-recursive-777` | DENY | `chmod -R 777 /` (escritura mundial recursiva en root) |
| Destructivo | `destructive-chown-recursive` | DENY | `chown -R` en rutas del sistema (`/usr`, `/etc`, `/var`, etc.) |
| Credenciales | `cred-gnupg` | DENY | Acceso a `~/.gnupg/**` (llavero GnuPG) |
| Credenciales | `cred-pem-files` | DENY | `cat`, `cp`, `scp` etc. en archivos `.pem` / `.key` |
| Credenciales | `cred-kube-config` | DENY | Acceso a `~/.kube/config` |
| Credenciales | `cred-docker-config` | DENY | Acceso a `~/.docker/config.json` |
| Exfiltracin | `exfil-base64-to-curl` | DENY | Patrn de pipe `base64 \| curl` |
| Exfiltracin | `exfil-tar-to-curl` | DENY | Patrn de pipe `tar \| curl` (archivo a red) |
| RCE | `rce-python-pipe` | DENY | Patrn de pipe `curl \| python` |
| RCE | `rce-eval-command` | REQUIRE_APPROVAL | Uso de `eval` (puede ejecutar cdigo arbitrario) |
| RCE | `rce-netcat-listener` | DENY | `nc -l` (potencial reverse shell) |
| Git | `dangerous-git-force-push` | DENY | `git push --force` a main/master |
| Paquetes | `dangerous-npm-publish` | REQUIRE_APPROVAL | `npm publish` |
| Contenedores | `dangerous-docker-privileged` | DENY | `docker run --privileged` |

## API de TypeScript

```typescript
import {
  getPackMetadata,
  loadBaselineRules,
  loadSessionTemplates,
  loadChannelDefaults,
  loadSkillDefaults,
} from "@securitylayerai/rules";

// Metadatos del paquete
const meta = getPackMetadata();
// { name: "@securitylayerai/rules", version: "0.0.1", type: "rule-pack", ruleCount: 14 }

// Cargar baseline rules como cadena YAML sin procesar
const rulesYaml = await loadBaselineRules();

// Cargar capability templates parseadas
const sessions = await loadSessionTemplates();
const channels = await loadChannelDefaults();
const skills = await loadSkillDefaults();
```

## Escribir rules personalizadas

Las baseline rules son un punto de partida. Puedes agregar tus propias rules usando el mismo formato YAML:

```yaml
version: 1
rules:
  - id: my-custom-rule
    description: Block access to production database
    match:
      type: pattern
      value: "\\bpsql\\s+.*production"
    decision: DENY
    reason: "Direct access to production database"
```

Tipos de coincidencia de rules:
- `pattern` -- Coincidencia regex contra la cadena completa del comando
- `path` -- Coincidencia glob contra rutas de archivos
- `pipe_pair` -- Coincidencia de estructuras de comandos encadenados (`from`  `to`)
