AI 에이전트가 시도하는 모든 동작은 3계층 보안 파이프라인을 통과합니다. 각 레이어는 독립적이고 상호 보완적이며, 서로 다른 범주의 위협을 잡아냅니다.

## 레이어 1: 구조적 적용 (벽)

주요 방어 수단입니다. 구조적 적용은 위험한 동작을 탐지하려는 것이 아니라 **불가능하게** 만듭니다.

구성 요소:

- **Capability 게이트** — 순수 함수 집합 소속 검사. capability가 없으면 동작도 없습니다.
- **Taint 추적** — 세션 신뢰 수준이 사용 가능한 capability를 제한합니다.
- **동작 정규화** — 평가 전에 경로 해석, 인코딩 디코딩, 명령 체인 분리를 수행합니다.
- **게이트웨이 잠금** — 보안 프록시 자체를 우회하는 것을 방지합니다.
- **실행 샌드박싱** — 승인된 명령이 제한된 환경(네트워크 차단, 읽기 전용 FS, ulimit)에서 실행됩니다.
- **송신 스캔** — 발신 메시지에서 유출된 비밀, PII, 고엔트로피 데이터를 스캔합니다.

구조적 적용은 교묘한 프롬프트, 사회 공학, 프롬프트 인젝션으로 우회할 수 없습니다.

## 레이어 2: 의미 분석 (경비)

LLM 심판이 구조적으로 허용된 동작이 **맥락적으로 비정상적인지** 분석합니다.

심판의 역할은 좁습니다 — 다음은 수행하지 않습니다:
- 인젝션 패턴 탐지 (taint 추적이 처리)
- 알려진 위험 명령 매칭 (규칙이 처리)

오직 다음만 질문합니다: **"이 구조적으로 허용된 동작이 지금 비정상적인가?"**

심판은 규칙 엔진과 **병렬로** 실행됩니다. 심판이 느리면(500ms 초과), 레이어 1 + 레이어 3이 즉시 결정하고 심판의 결과는 비동기 사후 검토에 사용됩니다.

심판의 결정:
- `NORMAL` — 동작이 예상된 것으로 보임
- `ANOMALOUS` — 동작이 비정상적임 → `REQUIRE_APPROVAL`
- `DANGEROUS` (신뢰도 >= 0.8) → `DENY`

심판을 사용할 수 없고 위험 점수가 0.3을 초과하면 승인이 필요합니다.

## 레이어 3: 결정적 규칙 (경보)

알려진 위험 명령 패턴 및 경로 차단. 빠르고, 지연 시간 제로, 모호성 제로.

규칙은 YAML로 정의되며 런타임에 로드됩니다:

```yaml
# 기본 규칙 예시
version: 1
rules:
  - pattern: "rm -rf /"
    action: DENY
    reason: "Recursive delete of root"
  - pattern: "chmod 777"
    action: DENY
    reason: "World-writable permissions"
```

규칙은 항상 우선합니다: 규칙이 DENY라고 하면 다른 레이어의 결정과 관계없이 동작이 거부됩니다.

## 결정 병합

레이어 결정은 **가장 제한적인** 결과를 취합니다:

| 조건 | 결과 |
|---|---|
| 규칙 DENY | **DENY** (항상) |
| LLM DANGEROUS (신뢰도 >= 0.8) | **DENY** |
| LLM ANOMALOUS | **REQUIRE_APPROVAL** |
| LLM 사용 불가 + 위험 > 0.3 | **REQUIRE_APPROVAL** |
| 모든 레이어 통과 | **ALLOW** |

## 안전 장치 원칙

모든 실패는 시스템을 **더 제한적으로** 만들며, 절대 덜 제한적으로 만들지 않습니다:

- LLM 다운 → 사용자 승인 필요
- 설정 손상 → 시작 거부
- Taint 오류 → 최대 taint로 가정
- 정규화 실패 → 최대 위험으로 처리

## 파이프라인 흐름

```
동작 → Capability 게이트 → Taint 검사 → 정규화 → ┬─ 규칙 엔진 ──┬→ 병합 → 결정
                                                   └─ LLM 심판 ───┘
```

Capability 게이트와 taint 검사는 동기적이고 즉시 완료됩니다. 정규화는 규칙 엔진과 LLM 심판의 병렬 평가를 위해 동작을 준비합니다. 병합기는 가장 제한적인 결과를 취합니다.
