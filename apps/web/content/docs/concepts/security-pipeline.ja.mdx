AI エージェントが試みるすべてのアクションは、3 層のセキュリティパイプラインを通過します。各レイヤーは独立しており補完的で、それぞれ異なるカテゴリの脅威を捕捉します。

## レイヤー 1: 構造的適用（壁）

主要な防御です。構造的適用は、危険なアクションを検知するのではなく**不可能に**します。

コンポーネント:

- **ケイパビリティゲート** — 純粋関数によるセットメンバーシップチェック。ケイパビリティなし = アクションなし。
- **テイントトラッキング** — セッションの信頼レベルが利用可能なケイパビリティを制限します。
- **アクション正規化** — 評価前にパスの解決、エンコーディングのデコード、コマンドチェーンの分割を行います。
- **ゲートウェイロックダウン** — セキュリティプロキシ全体のバイパスを防止します。
- **Exec サンドボックス** — 承認されたコマンドは制限環境（ネットワークなし、読み取り専用 FS、ulimit）で実行されます。
- **Egress スキャニング** — 送信メッセージが秘密情報、PII、高エントロピーデータについてスキャンされます。

構造的適用は、巧妙なプロンプト、ソーシャルエンジニアリング、プロンプトインジェクションでは回避できません。

## レイヤー 2: セマンティック分析（警備員）

LLM ジャッジは、構造的に許可されたアクションが**コンテキスト上異常**かどうかを分析します。

ジャッジの役割は限定的です。以下は行いません:
- インジェクションパターンの検出（テイントトラッキングが担当）
- 既知の危険コマンドのマッチング（ルールが担当）

唯一の問い: **「この構造的に許可されたアクションは、現在のコンテキストで異常ですか？」**

ジャッジはルールエンジンと**並列**で実行されます。ジャッジが遅い場合（>500ms）、レイヤー 1 + レイヤー 3 が即座に判定し、ジャッジの結果は非同期の事後レビューに使用されます。

ジャッジの判定:
- `NORMAL` — アクションは想定通り
- `ANOMALOUS` — アクションが通常と異なる → `REQUIRE_APPROVAL`
- `DANGEROUS`（確信度 >= 0.8）→ `DENY`

ジャッジが利用不可でリスクスコアが 0.3 を超える場合、承認が必要になります。

## レイヤー 3: 決定論的ルール（警報）

既知の危険コマンドパターンとパスブロック。高速、ゼロレイテンシー、ゼロアンビギュイティ。

ルールは YAML で定義され、ランタイムに読み込まれます:

```yaml
# ベースラインルールの例
version: 1
rules:
  - pattern: "rm -rf /"
    action: DENY
    reason: "Recursive delete of root"
  - pattern: "chmod 777"
    action: DENY
    reason: "World-writable permissions"
```

ルールは常に優先されます: ルールが DENY なら、他のレイヤーの判定に関わらずアクションは拒否されます。

## 判定のマージ

レイヤーの判定は**最も制限的な**結果を取ることでマージされます:

| 条件 | 結果 |
|---|---|
| ルールが DENY | **DENY**（常に） |
| LLM が DANGEROUS（確信度 >= 0.8） | **DENY** |
| LLM が ANOMALOUS | **REQUIRE_APPROVAL** |
| LLM が利用不可 + リスク > 0.3 | **REQUIRE_APPROVAL** |
| すべてのレイヤーが通過 | **ALLOW** |

## フェイルセーフの原則

すべての障害はシステムを**より制限的に**し、決して緩くしません:

- LLM がダウン → 人間の承認を要求
- 設定が破損 → 起動を拒否
- テイントエラー → 最大テイントを仮定
- 正規化失敗 → 最大限危険として扱う

## パイプラインフロー

```
Action → Capability Gate → Taint Check → Normalize → ┬─ Rules Engine ──┬→ Merge → Decision
                                                      └─ LLM Judge ─────┘
```

ケイパビリティゲートとテイントチェックは同期的で即座に完了します。正規化はルールエンジンと LLM ジャッジによる並列評価のためにアクションを準備します。マージャーが最も制限的な結果を採用します。
