셸 shim은 셸 명령을 실행하는 **모든** AI 도구를 보호하는 범용 통합으로, 도구 측에서 별도의 통합 작업이 필요하지 않습니다.

## 동작 원리

`sl shield enable`은 `~/.securitylayer/bin/`에 PATH 우선순위 래퍼 스크립트를 설치합니다. 이 디렉토리가 PATH 앞에 추가되므로, 모든 셸 명령이 실제 바이너리에 도달하기 전에 Security Layer를 통과합니다.

### shim 바이너리

| 바이너리 | 설명 |
|---|---|
| `bash` | Bash 셸 |
| `sh` | POSIX 셸 |
| `zsh` | Zsh 셸 |
| `python` | Python 인터프리터 |
| `python3` | Python 3 인터프리터 |
| `node` | Node.js 런타임 |
| `ruby` | Ruby 인터프리터 |
| `perl` | Perl 인터프리터 |

### 실행 흐름

```
AI 도구 → shim bash → sl check → 실제 bash
                            ↓ (거부 시)
                       오류 메시지
```

1. AI 도구가 `bash -c "some command"`를 호출합니다
2. `~/.securitylayer/bin/bash`에 있는 shim이 가로챕니다
3. `sl check`가 명령을 평가합니다
4. 허용 시 → 실제 `/usr/bin/bash`로 전달됩니다
5. 거부 시 → 오류가 반환되고, 명령은 절대 실행되지 않습니다

### 호출자 감지

shim은 다음을 확인하여 어떤 AI 도구가 명령을 트리거했는지 식별합니다:

- 각 도구에 고유한 환경 변수
- 부모 프로세스 이름

이를 통해 도구마다 다른 capability 프로필을 적용할 수 있습니다 (예: Claude Code는 `exec` + `file.write`를 부여받고, 신뢰하지 않는 도구는 `file.read`만 부여).

## 설정

```bash
# 활성화
sl shield enable

# 확인
sl shield status

# 비활성화
sl shield disable
```

활성화 후, PATH 변경이 적용되도록 셸을 재시작하거나 새 터미널을 열어야 합니다.

## 지원되는 AI 도구

셸 명령을 실행하는 모든 도구가 자동으로 보호됩니다:

- Claude Code
- Cursor
- Aider
- GitHub Copilot
- 커스텀 에이전트
- 셸 명령을 실행하는 모든 도구
