ラーニングモードは、実際にはブロックせずに Security Layer が**ブロックするはずだった**内容をモニタリングします。ポリシーを適用する前に使用パターンを把握するために使用します。

## 使い方

```bash
securitylayer learn [--duration <duration>]
```

## フラグ

| フラグ | デフォルト | 説明 |
|---|---|---|
| `--duration <duration>` | `7d` | ラーニングモードの継続期間 |

### 期間の形式

| サフィックス | 単位 | 例 |
|---|---|---|
| `m` | 分 | `30m` |
| `h` | 時間 | `1h`, `12h` |
| `d` | 日 | `1d`, `7d`, `30d` |

## 動作

ラーニングモードが有効な場合:

- すべてのアクションが**許可**されます（既存のケイパビリティセットは変更されません）
- 推奨設定で**ブロックされるはずだった**内容が Security Layer によってログに記録されます
- 指定した期間が経過すると、モードは自動的に終了します
- 設定に `mode: learning` と `learning_expires` タイムスタンプが記録されます

ラーニング期間終了後、`securitylayer learn report` を使用して、実際の使用状況に基づいた推奨ケイパビリティ変更を確認できます。

## 例

```bash
# 7 日間モニタリング
securitylayer learn --duration 7d

# ログされた内容を確認
securitylayer learn report
```

## 終了コード

| コード | 意味 |
|---|---|
| `0` | ラーニングモードが有効化 |
| `1` | 無効な期間形式 |

## なぜラーニングモードが必要か

ユーザーがセキュリティツールを無効にする最大の理由: 「インストールしたら全部壊れた」。ラーニングモードは、適用前にまず観察することでこの問題を防ぎます。
