프로젝트 신뢰 규칙, 현재 taint 레벨을 확인하고, taint 상태를 초기화합니다.

## 사용법

```bash
securitylayer taint show
securitylayer taint clear
```

## 하위 명령

### `taint show`

프로젝트 신뢰 규칙과 현재 작업 디렉토리의 taint 레벨을 표시합니다.

```bash
$ securitylayer taint show

  Project trust rules:
    ~/Dev/Personal/** → OWNER
    ~/Dev/Work/**     → TRUSTED_CONTACT
    /tmp/**           → WEB_CONTENT
    ~/Downloads/**    → WEB_CONTENT

  Default taint: UNTRUSTED_HUMAN

  Current directory: ~/Dev/Personal/myproject
  Current taint:    OWNER
```

### `taint clear`

CLI 모드에서 taint는 프로젝트 신뢰 규칙에 따라 결정되며 명령 간에 누적되지 않습니다. 이 명령은 taint 관리에 대한 안내를 제공합니다:

- 프로젝트 수준의 신뢰 변경은 `securitylayer projects trust <path>`를 사용합니다
- 데몬 모드(프록시)에서 taint 초기화는 세션의 누적된 taint를 리셋합니다

## 종료 코드

| 코드 | 의미 |
|---|---|
| `0` | 항상 성공적으로 종료됨 |
