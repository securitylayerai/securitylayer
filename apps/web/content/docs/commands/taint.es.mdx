Visualiza las reglas de confianza de proyectos, los niveles de taint actuales y limpia el estado de taint.

## Uso

```bash
sl taint show
sl taint clear
```

## Subcomandos

### `taint show`

Muestra las reglas de confianza de proyectos y el nivel de taint del directorio de trabajo actual.

```bash
$ sl taint show

  Project trust rules:
    ~/Dev/Personal/** → OWNER
    ~/Dev/Work/**     → TRUSTED_CONTACT
    /tmp/**           → WEB_CONTENT
    ~/Downloads/**    → WEB_CONTENT

  Default taint: UNTRUSTED_HUMAN

  Current directory: ~/Dev/Personal/myproject
  Current taint:    OWNER
```

### `taint clear`

En modo CLI, el taint se determina por las reglas de confianza de proyectos y no se acumula entre comandos. Este comando proporciona orientación sobre la gestión de taint:

- Para cambios de confianza a nivel de proyecto, usa `sl projects trust <path>`
- En modo daemon (proxy), limpiar el taint restablece el taint acumulado de la sesión

## Códigos de salida

| Código | Significado |
|---|---|
| `0` | Siempre termina exitosamente |
