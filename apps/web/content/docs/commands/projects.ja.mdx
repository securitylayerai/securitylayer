プロジェクト信頼ルールは、ディレクトリパスにテイントレベルを割り当てます。AI エージェントがディレクトリ内で操作する際、信頼ルールがセッションの初期テイントレベルを決定します。

## 使い方

```bash
sl projects list
sl projects trust <path> [--taint <level>]
sl projects untrust <path>
```

## サブコマンド

### `projects list`

設定済みのすべてのプロジェクト信頼ルールを表示します。

```bash
$ sl projects list

  Trust rules:
    1. ~/Dev/Personal/** → OWNER
    2. ~/Dev/Work/**     → TRUSTED_CONTACT
    3. /tmp/**           → WEB_CONTENT

  Default taint: UNTRUSTED_HUMAN
```

### `projects trust <path>`

ディレクトリパスの信頼ルールを追加または更新します。

```bash
sl projects trust ~/Dev/Personal/** --taint owner
```

| フラグ | デフォルト | 説明 |
|---|---|---|
| `--taint <level>` | `owner` | 信頼レベル: `owner`, `trusted`, `untrusted`, `web`, `skill`, `memory` |

**終了コード:** 成功時 `0`、パス引数が不足している場合 `1`。

### `projects untrust <path>`

ディレクトリパスの信頼ルールを削除します。

```bash
$ sl projects untrust /tmp/**
Removed trust rule for: /tmp/**
```

**終了コード:** 常に `0`、パス引数が不足している場合 `1`。
