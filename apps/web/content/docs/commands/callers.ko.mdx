Security Layer는 명령을 트리거한 AI 도구를 자동으로 감지하고 적절한 capability 프로필을 적용합니다. callers 명령으로 이러한 프로필을 확인할 수 있습니다.

## 사용법

```bash
securitylayer callers list
securitylayer callers profile <name>
```

## 하위 명령

### `callers list`

설정된 모든 AI 도구 호출자와 감지 규칙 및 capability를 표시합니다.

```bash
$ securitylayer callers list

  claude-code (Claude Code)
    taint: OWNER
    capabilities: exec, file.read, file.write, web_fetch
    detect env: CLAUDE_CODE
    detect process: claude

  cursor (Cursor)
    taint: TRUSTED_CONTACT
    capabilities: exec, file.read, file.write
    detect env: CURSOR_SESSION
    detect process: cursor
```

### `callers profile <name>`

특정 호출자의 상세 정보를 표시합니다.

```bash
$ securitylayer callers profile claude-code

  ID:           claude-code
  Display name: Claude Code
  Default taint: OWNER

  Capabilities:
    exec, file.read, file.write, web_fetch

  Detection:
    Environment: CLAUDE_CODE
    Process:     claude
```

**종료 코드:** 성공 시 `0`, 알 수 없는 호출자이거나 이름 인자가 누락된 경우 `1`.

## 감지 방식

명령이 가로채질 때(셸 shim 또는 훅을 통해), Security Layer는 다음을 확인합니다:

1. **환경 변수** — 호출 프로세스에 특정 환경 변수가 설정되어 있는가?
2. **프로세스 이름** — 부모 프로세스가 알려진 AI 도구와 일치하는가?

감지된 호출자에 따라 적용할 capability 프로필이 결정됩니다.
