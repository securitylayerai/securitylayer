이 가이드를 통해 5분 만에 제로 상태에서 보호 상태까지 설정할 수 있습니다.

## 1. 설치

```bash
bun add -g securitylayer
```

## 2. 초기화

```bash
sl init
```

기본값을 수락하거나 세션 capability를 커스터마이즈합니다. 마법사가 `~/.securitylayer/`에 설정 파일을 생성합니다.

## 3. 통합 설정

**Claude Code** — 네이티브 훅 통합:

```bash
sl setup claude-code
```

이 명령은 `~/.claude/hooks.json`에 PreToolUse 및 PostToolUse 훅을 설치합니다. Claude Code가 수행하는 모든 도구 호출이 실행 전에 보안 정책에 따라 평가됩니다.

**셸 shim** — 모든 AI 도구를 위한 범용 보호:

```bash
sl shield enable
```

이 명령은 모든 셸 명령을 가로채는 PATH 우선순위 래퍼 스크립트를 설치합니다. Cursor, Aider, Copilot 또는 셸 명령을 실행하는 모든 AI 도구에서 동작합니다.

## 4. 확인

```bash
sl status
```

설정이 로드되고, 세션 capability와 통합 상태(훅 설치 또는 shield 활성화)가 표시됩니다.

## 5. 테스트

dry-run 정책 검사를 실행하여 Security Layer의 동작을 확인합니다:

```bash
sl policy check "rm -rf /"
```

전체 평가 결과 — capability 게이트, 규칙 엔진, 위험 점수 — 와 최종 결정이 표시됩니다.

## 동작 원리

AI 에이전트가 명령을 실행하려고 할 때:

1. **Capability 게이트** — 이 동작 유형이 허용되어 있습니까? (`exec`, `file.write` 등)
2. **Taint 검사** — 세션 신뢰 수준이 충분합니까?
3. **정규화** — 경로 해석, 인코딩 디코딩, 명령 체인 분리
4. **규칙 엔진** — 결정적 패턴 매칭
5. **LLM 심판** — 맥락적 이상 징후에 대한 의미 분석 (활성화된 경우)
6. **결정** — 가장 제한적인 결과가 적용됩니다: ALLOW, DENY, 또는 REQUIRE_APPROVAL
