---
title: ラーニングモード
description: 適用前に Security Layer がブロックする内容をモニタリングします。
icon: Brain
---

ラーニングモードは、ワークフローに影響を与えずに Security Layer の判定を観察できます。すべてが許可されますが、ブロックされるはずだったアクションがログに記録されるため、適用前にポリシーを調整できます。

## 有効化

```bash
securitylayer learn --duration 7d
```

設定に `mode: learning` が設定され、`learning_expires` タイムスタンプが記録されます。期間が経過すると、Security Layer は適用モードに戻ります。

## ログされる内容

ラーニングモードでは、Security Layer はすべてのアクションをセキュリティパイプライン全体で評価し、以下をログに記録します:

- **拒否されるはずだった**アクションとその理由
- ブロックをトリガーしたレイヤー（ケイパビリティゲート、ルール、LLM ジャッジ）
- 各レイヤーのタイミング情報
- 呼び出し元の識別子とテイントレベル

## ステータスの確認

```bash
securitylayer status
```

ラーニングモードが有効かどうか、有効期限、ログされたアクションの数が表示されます。

## 結果の確認

ラーニング期間後:

```bash
securitylayer learn report
```

以下が表示されます:

- **ブロックされるはずだったアクション** — 理由別にグループ化
- **推奨ケイパビリティ変更** — 実際の使用パターンに基づく
- **推奨ルール** — 頻繁に現れたパターン

## ワークフロー

1. **インストールと初期化** — `securitylayer init`
2. **ラーニング有効化** — `securitylayer learn --duration 7d`
3. **通常通り作業** — AI ツールをいつも通り使用
4. **レビュー** — `securitylayer learn report`
5. **ポリシー調整** — 結果に基づいてケイパビリティとルールを調整
6. **適用** — ラーニングモードが終了し、適用が開始

## ヒント

- ワークフローの代表的なサンプルを取得するには 7 日間から始めましょう
- 素早く反復したい場合は短い期間（1-2 日）を使用してください
- ラーニング期間が終了する前にレポートを確認して調整しましょう
- `config.yaml` を編集して `mode: learning` の行を削除すれば、早期にラーニングモードを無効化できます
