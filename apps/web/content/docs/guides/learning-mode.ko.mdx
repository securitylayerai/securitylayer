학습 모드는 워크플로에 영향을 주지 않으면서 Security Layer의 결정을 관찰할 수 있게 합니다. 모든 것이 허용되지만, 차단될 동작이 로깅되어 적용 전에 정책을 조정할 수 있습니다.

## 활성화

```bash
securitylayer learn --duration 7d
```

이 명령은 설정에 `mode: learning`을 설정하고 `learning_expires` 타임스탬프를 기록합니다. 기간이 지나면 Security Layer가 적용 모드로 복귀합니다.

## 로깅되는 항목

학습 모드에서 Security Layer는 전체 보안 파이프라인을 통해 모든 동작을 평가하고 다음을 로깅합니다:

- **거부되었을 동작**과 그 이유
- 차단을 트리거한 레이어 (capability 게이트, 규칙, LLM 심판)
- 각 레이어의 타이밍 정보
- 호출자 ID 및 taint 레벨

## 상태 확인

```bash
securitylayer status
```

학습 모드의 활성 여부, 만료 시점, 로깅된 동작 수를 표시합니다.

## 결과 검토

학습 기간이 끝난 후:

```bash
securitylayer learn report
```

다음을 표시합니다:

- **차단되었을 동작** — 이유별로 그룹화
- **권장 capability 변경사항** — 실제 사용 패턴 기반
- **제안 규칙** — 자주 나타난 패턴

## 워크플로

1. **설치 및 초기화** — `securitylayer init`
2. **학습 활성화** — `securitylayer learn --duration 7d`
3. **평소대로 작업** — AI 도구를 평소처럼 사용
4. **검토** — `securitylayer learn report`
5. **정책 조정** — 결과에 따라 capability와 규칙 조정
6. **적용** — 학습 모드가 만료되면 적용 시작

## 팁

- 워크플로의 대표적인 샘플을 수집하기 위해 7일로 시작합니다
- 빠르게 반복하려면 더 짧은 기간(1-2일)을 사용합니다
- 학습 기간이 끝나기 전에 보고서를 검토하여 조정합니다
- `config.yaml`을 편집하고 `mode: learning` 줄을 제거하면 학습 모드를 일찍 비활성화할 수 있습니다
