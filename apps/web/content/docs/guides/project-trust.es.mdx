## Configurar reglas de confianza

### Via CLI

```bash

# Confiar en proyectos de trabajo a un nivel menor
securitylayer projects trust "~/Dev/Work/**" --taint trusted

# Marcar directorios temporales como no confiables

# Eliminar una regla de confianza

Edita `~/.securitylayer/projects.yaml`:

  - path: "~/Dev/Personal/**"

  - path: "~/Dev/Work/**"

```

## Mapeo de niveles de taint

| Valor CLI | Nivel de taint | Caso de uso |
|---|---|---|
| `owner` | `OWNER` | Tus propios proyectos |
| `trusted` | `TRUSTED_CONTACT` | Proyectos de colaboradores conocidos |
| `untrusted` | `UNTRUSTED_HUMAN` | Fuentes desconocidas |
| `web` | `WEB_CONTENT` | Descargados/clonados de internet |
| `skill` | `SKILL_GENERATED` | Generados por skills |
| `memory` | `MEMORY_REPLAY` | De memoria persistente |
