## createSecurityLayer()

Crea y retorna una instancia de `SecurityLayer`. Este es el punto de entrada principal del SDK.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer(options?: SecurityLayerOptions);
```

### Opciones

| Opcion | Tipo | Valor por defecto | Descripcion |
|---|---|---|---|
| `configDir` | `string` | `~/.securitylayer` | Ruta al directorio de configuracion de Security Layer |
| `sessionId` | `string` | UUID auto-generado | ID de sesion por defecto para verificaciones |
| `config` | `LoadedConfig` | â€” | Objeto de configuracion pre-cargado (omite la carga de archivos, util para testing) |
| `eventBus` | `EventBus` | Auto-creado | Event bus inyectado (util para testing o compartir eventos) |

**Lanza:** `ConfigError` si los archivos de configuracion no se pueden cargar, `InitializationError` si el security pipeline falla al inicializarse.

## SecurityLayer

El objeto retornado por `createSecurityLayer()`. Proporciona todos los metodos de verificacion de seguridad, taint tracking y manejo de eventos.

### check()

Evalua una accion a traves del security pipeline antes de su ejecucion.

```typescript
const result = await sl.check(
  tool: string,
  params: Record<string, unknown>,
  context?: ExecutionContext,
): Promise<CheckResult>;
```

**Parametros:**

| Parametro | Tipo | Descripcion |
|---|---|---|
| `tool` | `string` | La capability que se esta usando: `"exec"`, `"file.read"`, `"file.write"`, `"web_fetch"`, etc. |
| `params` | `object` | Parametros especificos de la herramienta. Para exec, el comando se extrae de las propiedades `command`, `cmd` o `input`. |
| `context` | `ExecutionContext` | Sobreescrituras opcionales de contexto para esta verificacion |

**Retorna:** `Promise<CheckResult>`

### waitForApproval()

Bloquea hasta que un humano apruebe o deniegue una decision `REQUIRE_APPROVAL`.

```typescript
const approved = await sl.waitForApproval(
  approvalId: string,
  options?: ApprovalOptions,
): Promise<boolean>;
```

**Parametros:**

| Parametro | Tipo | Descripcion |
|---|---|---|
| `approvalId` | `string` | El ID de aprobacion de un resultado `REQUIRE_APPROVAL` |
| `options.timeout` | `number` | Tiempo limite en milisegundos (por defecto: 300000 / 5 minutos) |

**Retorna:** `true` si fue aprobado, `false` si fue denegado, se agoto el tiempo, o el `approvalId` es desconocido.

### ingestContent()

Actualiza el rastreador de taint cuando contenido externo entra en la sesion.

```typescript
sl.ingestContent(
  content: string,
  metadata: ContentMetadata,
): void;
```

**Parametros:**

| Parametro | Tipo | Descripcion |
|---|---|---|
| `content` | `string` | El contenido que fue leido u obtenido |
| `metadata.source` | `"web" \| "file" \| "channel" \| "skill" \| "memory"` | De donde proviene el contenido |
| `metadata.path` | `string?` | Ruta del archivo, nombre del skill o identificador del canal |
| `metadata.url` | `string?` | URL (para fuentes web) |

**Mapeo de source a taint level:**

| Source | Taint Level |
|---|---|
| `web` | WEB |
| `file` | UNTRUSTED |
| `channel` | UNTRUSTED |
| `skill` | SKILL |
| `memory` | MEMORY |

### scanEgress()

Escanea contenido saliente en busca de secretos filtrados, PII y datos sensibles antes de enviarlos.

```typescript
const result = sl.scanEgress(content: string): EgressScanResult;
```

**Retorna:** `EgressScanResult` con `clean: boolean` y `findings: EgressFinding[]`.

### getSessionTaint()

Obtiene el taint level efectivo de la sesion actual.

```typescript
const taint = sl.getSessionTaint(): string;
```

**Retorna:** Uno de `"owner"`, `"trusted"`, `"untrusted"`, `"web"`, `"skill"`, `"memory"`.

### on()

Se suscribe a eventos de seguridad del event bus interno.

```typescript
const unsubscribe = sl.on(type: string, handler: (event) => void): () => void;
```

**Parametros:**

| Parametro | Tipo | Descripcion |
|---|---|---|
| `type` | `string` | Tipo de evento a escuchar (ej., `"action.evaluated"`, `"taint.elevated"`, `"approval.requested"`) |
| `handler` | `function` | Callback invocado cuando el evento se dispara |

**Retorna:** Una funcion para cancelar la suscripcion. Llamala para dejar de recibir eventos.

### destroy()

Limpia recursos. Resuelve todas las solicitudes de aprobacion pendientes como expiradas y limpia todos los manejadores de eventos.

```typescript
sl.destroy(): void;
```

Siempre llama a `destroy()` cuando hayas terminado con la instancia de SecurityLayer.

## withSecurityLayer()

Middleware wrapper que agrega verificaciones de seguridad a cualquier funcion. Intercepta llamadas de forma transparente -- la funcion envuelta tiene la misma firma que la original.

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

const protectedExec = withSecurityLayer(sl, originalExecFn, "exec");
const protectedRead = withSecurityLayer(sl, originalReadFn, "file.read");
```

**Firma:**

```typescript
function withSecurityLayer<TArgs extends unknown[], TReturn>(
  sl: SecurityLayer,
  fn: (...args: TArgs) => TReturn | Promise<TReturn>,
  tool: string,
  options?: {
    extractParams?: (...args: unknown[]) => Record<string, unknown>;
  },
): (...args: TArgs) => Promise<TReturn>;
```

**Comportamiento:**
- `ALLOW` -> llama a la funcion original y retorna su resultado
- `DENY` -> lanza `SecurityLayerError` sin llamar a la funcion
- `REQUIRE_APPROVAL` -> espera la aprobacion, lanza error si es denegada

**Extraccion de parametros por defecto:** Si el primer argumento es un string, se mapea a `{ command: arg }`. Si es un objeto, se usa directamente como parametros.

## Clases de error

Todos los errores del SDK extienden `SecurityLayerError`, que a su vez extiende `Error`.

```typescript
import {
  SecurityLayerError,
  ConfigError,
  InitializationError,
  CheckError,
  ApprovalTimeoutError,
} from "@securitylayerai/sdk";
```

| Error | Propiedades | Se lanza cuando |
|---|---|---|
| `SecurityLayerError` | `message` | Clase base; tambien lanzado por `withSecurityLayer()` en DENY |
| `ConfigError` | `configPath?: string` | Los archivos de configuracion no se pueden cargar o parsear |
| `InitializationError` | `cause?: unknown` | El security pipeline falla al inicializarse |
| `CheckError` | `tool?: string` | La evaluacion de una accion falla inesperadamente |
| `ApprovalTimeoutError` | `approvalId: string` | La solicitud de aprobacion expira |

## Tipos

### CheckResult

```typescript
interface CheckResult {
  decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL";
  reason: string;
  approvalId?: string;
  layers: LayerResults;
  degraded: boolean;
  timing: {
    total: number;
    capability?: number;
    rules?: number;
    llm?: number;
  };
}
```

### LayerResults

```typescript
interface LayerResults {
  capability: {
    allowed: boolean;
    reason?: string;
  };
  rules?: {
    decision: string;
    reason?: string;
  };
  llm?: {
    decision: string;
    confidence: number;
    reasoning?: string;
  };
}
```

### EgressScanResult

```typescript
interface EgressScanResult {
  clean: boolean;
  findings: EgressFinding[];
}

interface EgressFinding {
  type:
    | "api_key"
    | "pem_block"
    | "private_key"
    | "password"
    | "high_entropy"
    | "pii"
    | "credential_pattern";
  confidence: number;
  location: { start: number; end: number };
  redacted: string;
}
```

### ExecutionContext

```typescript
interface ExecutionContext {
  sessionId?: string;
  skillId?: string;
  channelId?: string;
}
```

### ContentMetadata

```typescript
interface ContentMetadata {
  source: "web" | "file" | "channel" | "skill" | "memory";
  path?: string;
  url?: string;
}
```

### SecurityLayerOptions

```typescript
interface SecurityLayerOptions {
  configDir?: string;
  sessionId?: string;
  config?: LoadedConfig;
  eventBus?: EventBus;
}
```

### ApprovalOptions

```typescript
interface ApprovalOptions {
  timeout?: number; // milliseconds, default 300000 (5 min)
}
```
