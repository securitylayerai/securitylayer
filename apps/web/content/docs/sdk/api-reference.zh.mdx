## createSecurityLayer()

创建并返回一个 `SecurityLayer` 实例。这是 SDK 的主入口。

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer(options?: SecurityLayerOptions);
```

### 选项

| 选项 | 类型 | 默认值 | 说明 |
|---|---|---|---|
| `configDir` | `string` | `~/.securitylayer` | Security Layer 配置目录的路径 |
| `sessionId` | `string` | 自动生成的 UUID | 检查时使用的默认 session ID |
| `config` | `LoadedConfig` | — | 预加载的配置对象（跳过文件加载，适用于测试） |
| `eventBus` | `EventBus` | 自动创建 | 注入的事件总线（适用于测试或共享事件） |

**抛出异常：** 如果无法加载配置文件则抛出 `ConfigError`，如果 security pipeline 初始化失败则抛出 `InitializationError`。

## SecurityLayer

`createSecurityLayer()` 返回的对象。提供所有安全检查、taint tracking 和事件处理方法。

### check()

在执行操作前，通过 security pipeline 评估该操作。

```typescript
const result = await sl.check(
  tool: string,
  params: Record<string, unknown>,
  context?: ExecutionContext,
): Promise<CheckResult>;
```

**参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `tool` | `string` | 正在使用的 capability：`"exec"`、`"file.read"`、`"file.write"`、`"web_fetch"` 等 |
| `params` | `object` | 工具特定的参数。对于 exec，命令从 `command`、`cmd` 或 `input` 属性中提取。 |
| `context` | `ExecutionContext` | 此次检查的可选上下文覆盖 |

**返回值：** `Promise<CheckResult>`

### waitForApproval()

阻塞等待，直到人工批准或拒绝一个 `REQUIRE_APPROVAL` 决策。

```typescript
const approved = await sl.waitForApproval(
  approvalId: string,
  options?: ApprovalOptions,
): Promise<boolean>;
```

**参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `approvalId` | `string` | 来自 `REQUIRE_APPROVAL` 检查结果的审批 ID |
| `options.timeout` | `number` | 超时时间（毫秒）（默认：300000 / 5 分钟） |

**返回值：** 批准时返回 `true`，拒绝、超时或 `approvalId` 未知时返回 `false`。

### ingestContent()

当外部内容进入会话时，更新 taint 追踪器。

```typescript
sl.ingestContent(
  content: string,
  metadata: ContentMetadata,
): void;
```

**参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `content` | `string` | 被读取或获取的内容 |
| `metadata.source` | `"web" \| "file" \| "channel" \| "skill" \| "memory"` | 内容来源 |
| `metadata.path` | `string?` | 文件路径、skill 名称或 channel 标识符 |
| `metadata.url` | `string?` | URL（用于 web 来源） |

**来源到 taint 的映射：**

| 来源 | Taint Level |
|---|---|
| `web` | WEB |
| `file` | UNTRUSTED |
| `channel` | UNTRUSTED |
| `skill` | SKILL |
| `memory` | MEMORY |

### scanEgress()

在发送之前，扫描出站内容中的泄露密钥、PII 和敏感数据。

```typescript
const result = sl.scanEgress(content: string): EgressScanResult;
```

**返回值：** `EgressScanResult`，包含 `clean: boolean` 和 `findings: EgressFinding[]`。

### getSessionTaint()

获取当前会话的有效 taint level。

```typescript
const taint = sl.getSessionTaint(): string;
```

**返回值：** `"owner"`、`"trusted"`、`"untrusted"`、`"web"`、`"skill"`、`"memory"` 之一。

### on()

订阅来自内部事件总线的安全事件。

```typescript
const unsubscribe = sl.on(type: string, handler: (event) => void): () => void;
```

**参数：**

| 参数 | 类型 | 说明 |
|---|---|---|
| `type` | `string` | 要监听的事件类型（如 `"action.evaluated"`、`"taint.elevated"`、`"approval.requested"`） |
| `handler` | `function` | 事件触发时调用的回调函数 |

**返回值：** 一个取消订阅函数。调用它可以停止接收事件。

### destroy()

清理资源。将所有待处理的审批请求标记为超时，并清除所有事件处理器。

```typescript
sl.destroy(): void;
```

使用完 SecurityLayer 实例后，务必调用 `destroy()`。

## withSecurityLayer()

中间件包装器，可以为任意函数添加安全检查。透明地拦截调用——被包装的函数与原始函数具有相同的签名。

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

const protectedExec = withSecurityLayer(sl, originalExecFn, "exec");
const protectedRead = withSecurityLayer(sl, originalReadFn, "file.read");
```

**签名：**

```typescript
function withSecurityLayer<TArgs extends unknown[], TReturn>(
  sl: SecurityLayer,
  fn: (...args: TArgs) => TReturn | Promise<TReturn>,
  tool: string,
  options?: {
    extractParams?: (...args: unknown[]) => Record<string, unknown>;
  },
): (...args: TArgs) => Promise<TReturn>;
```

**行为：**
- `ALLOW` → 调用原始函数并返回其结果
- `DENY` → 抛出 `SecurityLayerError`，不调用原始函数
- `REQUIRE_APPROVAL` → 等待审批，如果被拒绝则抛出异常

**默认参数提取：** 如果第一个参数是字符串，则映射为 `{ command: arg }`。如果是对象，则直接作为参数使用。

## 错误类

所有 SDK 错误都继承自 `SecurityLayerError`，后者继承自 `Error`。

```typescript
import {
  SecurityLayerError,
  ConfigError,
  InitializationError,
  CheckError,
  ApprovalTimeoutError,
} from "@securitylayerai/sdk";
```

| 错误 | 属性 | 触发条件 |
|---|---|---|
| `SecurityLayerError` | `message` | 基类；也由 `withSecurityLayer()` 在 DENY 时抛出 |
| `ConfigError` | `configPath?: string` | 无法加载或解析配置文件 |
| `InitializationError` | `cause?: unknown` | Security pipeline 初始化失败 |
| `CheckError` | `tool?: string` | 操作评估出现意外错误 |
| `ApprovalTimeoutError` | `approvalId: string` | 审批请求超时 |

## 类型

### CheckResult

```typescript
interface CheckResult {
  decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL";
  reason: string;
  approvalId?: string;
  layers: LayerResults;
  degraded: boolean;
  timing: {
    total: number;
    capability?: number;
    rules?: number;
    llm?: number;
  };
}
```

### LayerResults

```typescript
interface LayerResults {
  capability: {
    allowed: boolean;
    reason?: string;
  };
  rules?: {
    decision: string;
    reason?: string;
  };
  llm?: {
    decision: string;
    confidence: number;
    reasoning?: string;
  };
}
```

### EgressScanResult

```typescript
interface EgressScanResult {
  clean: boolean;
  findings: EgressFinding[];
}

interface EgressFinding {
  type:
    | "api_key"
    | "pem_block"
    | "private_key"
    | "password"
    | "high_entropy"
    | "pii"
    | "credential_pattern";
  confidence: number;
  location: { start: number; end: number };
  redacted: string;
}
```

### ExecutionContext

```typescript
interface ExecutionContext {
  sessionId?: string;
  skillId?: string;
  channelId?: string;
}
```

### ContentMetadata

```typescript
interface ContentMetadata {
  source: "web" | "file" | "channel" | "skill" | "memory";
  path?: string;
  url?: string;
}
```

### SecurityLayerOptions

```typescript
interface SecurityLayerOptions {
  configDir?: string;
  sessionId?: string;
  config?: LoadedConfig;
  eventBus?: EventBus;
}
```

### ApprovalOptions

```typescript
interface ApprovalOptions {
  timeout?: number; // milliseconds, default 300000 (5 min)
}
```
