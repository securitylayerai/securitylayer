## Por que usar o SDK?

As integrações via CLI (hooks, shell shim) funcionam sem alterações de código — mas operam na fronteira. O SDK oferece **segurança in-process** com contexto completo:

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| Intercepta exec | Sim | Sim | Sim |
| Intercepta escrita de arquivos | Não | Sim | Sim |
| Intercepta web fetch | Não | Sim | Sim |
| Contexto completo da sessão | Não | Parcial | **Sim** |
| Precisão do taint tracking | Grosseira | Moderada | **Por operação** |
| Risco de bypass | Manipulação de PATH | Nenhum | **Nenhum (in-process)** |
| Funciona com qualquer framework | Sim (apenas shell) | Apenas Claude Code | **Qualquer framework** |
| Esforço de integração | Zero | Um comando | Alterações no código |

O SDK é a escolha certa quando você está construindo uma aplicação agêntica e quer garantias de segurança estrutural — não apenas interceptação de linha de comando.

## Exemplo rápido

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

// Check an action before executing it
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`Blocked: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId as string);
  if (!approved) console.error("Approval denied");
}

// Clean up when done
sl.destroy();
```
