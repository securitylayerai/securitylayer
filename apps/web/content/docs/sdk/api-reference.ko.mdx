## createSecurityLayer()

`SecurityLayer` 인스턴스를 생성하여 반환합니다. SDK의 메인 진입점입니다.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer(options?: SecurityLayerOptions);
```

### 옵션

| 옵션 | 타입 | 기본값 | 설명 |
|---|---|---|---|
| `configDir` | `string` | `~/.securitylayer` | Security Layer 설정 디렉토리 경로 |
| `sessionId` | `string` | 자동 생성 UUID | 검사에 사용되는 기본 세션 ID |
| `config` | `LoadedConfig` | — | 미리 로드된 설정 객체 (파일 로딩 생략, 테스트에 유용) |
| `eventBus` | `EventBus` | 자동 생성 | 주입된 이벤트 버스 (테스트 또는 이벤트 공유에 유용) |

**Throws:** 설정 파일을 로드할 수 없으면 `ConfigError`, security pipeline 초기화에 실패하면 `InitializationError`를 던집니다.

## SecurityLayer

`createSecurityLayer()`가 반환하는 객체입니다. 모든 보안 검사, taint tracking, 이벤트 처리 메서드를 제공합니다.

### check()

실행 전에 security pipeline을 통해 동작을 평가합니다.

```typescript
const result = await sl.check(
  tool: string,
  params: Record<string, unknown>,
  context?: ExecutionContext,
): Promise<CheckResult>;
```

**매개변수:**

| 매개변수 | 타입 | 설명 |
|---|---|---|
| `tool` | `string` | 사용되는 capability: `"exec"`, `"file.read"`, `"file.write"`, `"web_fetch"` 등 |
| `params` | `object` | 도구별 매개변수. exec의 경우 `command`, `cmd` 또는 `input` 속성에서 명령을 추출합니다. |
| `context` | `ExecutionContext` | 이 검사에 대한 선택적 컨텍스트 오버라이드 |

**반환값:** `Promise<CheckResult>`

### waitForApproval()

사람이 `REQUIRE_APPROVAL` 결정을 승인하거나 거부할 때까지 대기합니다.

```typescript
const approved = await sl.waitForApproval(
  approvalId: string,
  options?: ApprovalOptions,
): Promise<boolean>;
```

**매개변수:**

| 매개변수 | 타입 | 설명 |
|---|---|---|
| `approvalId` | `string` | `REQUIRE_APPROVAL` 검사 결과에서 받은 승인 ID |
| `options.timeout` | `number` | 타임아웃(밀리초) (기본값: 300000 / 5분) |

**반환값:** 승인되면 `true`, 거부되거나 시간 초과되거나 `approvalId`를 알 수 없으면 `false`.

### ingestContent()

외부 콘텐츠가 세션에 유입될 때 taint 추적기를 업데이트합니다.

```typescript
sl.ingestContent(
  content: string,
  metadata: ContentMetadata,
): void;
```

**매개변수:**

| 매개변수 | 타입 | 설명 |
|---|---|---|
| `content` | `string` | 읽거나 가져온 콘텐츠 |
| `metadata.source` | `"web" \| "file" \| "channel" \| "skill" \| "memory"` | 콘텐츠의 출처 |
| `metadata.path` | `string?` | 파일 경로, 스킬 이름 또는 채널 식별자 |
| `metadata.url` | `string?` | URL (웹 소스인 경우) |

**소스-taint 매핑:**

| 소스 | Taint Level |
|---|---|
| `web` | WEB |
| `file` | UNTRUSTED |
| `channel` | UNTRUSTED |
| `skill` | SKILL |
| `memory` | MEMORY |

### scanEgress()

전송 전에 발신 콘텐츠에서 유출된 비밀, PII 및 민감한 데이터를 스캔합니다.

```typescript
const result = sl.scanEgress(content: string): EgressScanResult;
```

**반환값:** `clean: boolean`과 `findings: EgressFinding[]`이 포함된 `EgressScanResult`.

### getSessionTaint()

현재 세션의 유효 taint level을 가져옵니다.

```typescript
const taint = sl.getSessionTaint(): string;
```

**반환값:** `"owner"`, `"trusted"`, `"untrusted"`, `"web"`, `"skill"`, `"memory"` 중 하나.

### on()

내부 이벤트 버스에서 보안 이벤트를 구독합니다.

```typescript
const unsubscribe = sl.on(type: string, handler: (event) => void): () => void;
```

**매개변수:**

| 매개변수 | 타입 | 설명 |
|---|---|---|
| `type` | `string` | 수신할 이벤트 타입 (예: `"action.evaluated"`, `"taint.elevated"`, `"approval.requested"`) |
| `handler` | `function` | 이벤트 발생 시 호출되는 콜백 |

**반환값:** 구독 해제 함수. 호출하면 이벤트 수신이 중단됩니다.

### destroy()

리소스를 정리합니다. 모든 대기 중인 승인 요청을 시간 초과로 해결하고 모든 이벤트 핸들러를 제거합니다.

```typescript
sl.destroy(): void;
```

SecurityLayer 인스턴스 사용이 끝나면 항상 `destroy()`를 호출하세요.

## withSecurityLayer()

모든 함수에 보안 검사를 추가하는 미들웨어 래퍼입니다. 호출을 투명하게 가로채며 — 래핑된 함수는 원본과 동일한 시그니처를 갖습니다.

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

const protectedExec = withSecurityLayer(sl, originalExecFn, "exec");
const protectedRead = withSecurityLayer(sl, originalReadFn, "file.read");
```

**시그니처:**

```typescript
function withSecurityLayer<TArgs extends unknown[], TReturn>(
  sl: SecurityLayer,
  fn: (...args: TArgs) => TReturn | Promise<TReturn>,
  tool: string,
  options?: {
    extractParams?: (...args: unknown[]) => Record<string, unknown>;
  },
): (...args: TArgs) => Promise<TReturn>;
```

**동작:**
- `ALLOW` -> 원본 함수를 호출하고 결과를 반환합니다
- `DENY` -> 함수를 호출하지 않고 `SecurityLayerError`를 던집니다
- `REQUIRE_APPROVAL` -> 승인을 기다리고, 거부되면 에러를 던집니다

**기본 매개변수 추출:** 첫 번째 인자가 문자열이면 `{ command: arg }`로 매핑됩니다. 객체이면 직접 params로 사용됩니다.

## 에러 클래스

모든 SDK 에러는 `Error`를 확장하는 `SecurityLayerError`를 확장합니다.

```typescript
import {
  SecurityLayerError,
  ConfigError,
  InitializationError,
  CheckError,
  ApprovalTimeoutError,
} from "@securitylayerai/sdk";
```

| 에러 | 속성 | 발생 시점 |
|---|---|---|
| `SecurityLayerError` | `message` | 기본 클래스; `withSecurityLayer()`에서 DENY 시에도 던져짐 |
| `ConfigError` | `configPath?: string` | 설정 파일을 로드하거나 파싱할 수 없을 때 |
| `InitializationError` | `cause?: unknown` | Security pipeline 초기화에 실패할 때 |
| `CheckError` | `tool?: string` | 동작 평가가 예기치 않게 실패할 때 |
| `ApprovalTimeoutError` | `approvalId: string` | 승인 요청이 시간 초과될 때 |

## 타입

### CheckResult

```typescript
interface CheckResult {
  decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL";
  reason: string;
  approvalId?: string;
  layers: LayerResults;
  degraded: boolean;
  timing: {
    total: number;
    capability?: number;
    rules?: number;
    llm?: number;
  };
}
```

### LayerResults

```typescript
interface LayerResults {
  capability: {
    allowed: boolean;
    reason?: string;
  };
  rules?: {
    decision: string;
    reason?: string;
  };
  llm?: {
    decision: string;
    confidence: number;
    reasoning?: string;
  };
}
```

### EgressScanResult

```typescript
interface EgressScanResult {
  clean: boolean;
  findings: EgressFinding[];
}

interface EgressFinding {
  type:
    | "api_key"
    | "pem_block"
    | "private_key"
    | "password"
    | "high_entropy"
    | "pii"
    | "credential_pattern";
  confidence: number;
  location: { start: number; end: number };
  redacted: string;
}
```

### ExecutionContext

```typescript
interface ExecutionContext {
  sessionId?: string;
  skillId?: string;
  channelId?: string;
}
```

### ContentMetadata

```typescript
interface ContentMetadata {
  source: "web" | "file" | "channel" | "skill" | "memory";
  path?: string;
  url?: string;
}
```

### SecurityLayerOptions

```typescript
interface SecurityLayerOptions {
  configDir?: string;
  sessionId?: string;
  config?: LoadedConfig;
  eventBus?: EventBus;
}
```

### ApprovalOptions

```typescript
interface ApprovalOptions {
  timeout?: number; // milliseconds, default 300000 (5 min)
}
```
