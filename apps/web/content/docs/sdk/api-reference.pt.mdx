## createSecurityLayer()

Cria e retorna uma instância `SecurityLayer`. Este é o ponto de entrada principal do SDK.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer(options?: SecurityLayerOptions);
```

### Opções

| Opção | Tipo | Padrão | Descrição |
|---|---|---|---|
| `configDir` | `string` | `~/.securitylayer` | Caminho para o diretório de configuração do Security Layer |
| `sessionId` | `string` | UUID gerado automaticamente | ID de sessão padrão para verificações |
| `config` | `LoadedConfig` | — | Objeto de configuração pré-carregado (pula o carregamento de arquivos, útil para testes) |
| `eventBus` | `EventBus` | Criado automaticamente | Event bus injetado (útil para testes ou compartilhamento de eventos) |

**Lança:** `ConfigError` se os arquivos de configuração não puderem ser carregados, `InitializationError` se o security pipeline falhar na inicialização.

## SecurityLayer

O objeto retornado por `createSecurityLayer()`. Fornece todos os métodos de verificação de segurança, taint tracking e tratamento de eventos.

### check()

Avalia uma ação através do security pipeline antes da execução.

```typescript
const result = await sl.check(
  tool: string,
  params: Record<string, unknown>,
  context?: ExecutionContext,
): Promise<CheckResult>;
```

**Parâmetros:**

| Parâmetro | Tipo | Descrição |
|---|---|---|
| `tool` | `string` | A capability sendo utilizada: `"exec"`, `"file.read"`, `"file.write"`, `"web_fetch"`, etc. |
| `params` | `object` | Parâmetros específicos da ferramenta. Para exec, o comando é extraído das propriedades `command`, `cmd` ou `input`. |
| `context` | `ExecutionContext` | Sobrescritas de contexto opcionais para esta verificação |

**Retorna:** `Promise<CheckResult>`

### waitForApproval()

Bloqueia até que um humano aprove ou negue uma decisão `REQUIRE_APPROVAL`.

```typescript
const approved = await sl.waitForApproval(
  approvalId: string,
  options?: ApprovalOptions,
): Promise<boolean>;
```

**Parâmetros:**

| Parâmetro | Tipo | Descrição |
|---|---|---|
| `approvalId` | `string` | O ID de aprovação de um resultado de verificação `REQUIRE_APPROVAL` |
| `options.timeout` | `number` | Timeout em milissegundos (padrão: 300000 / 5 minutos) |

**Retorna:** `true` se aprovado, `false` se negado, expirado ou `approvalId` desconhecido.

### ingestContent()

Atualiza o rastreador de taint quando conteúdo externo entra na sessão.

```typescript
sl.ingestContent(
  content: string,
  metadata: ContentMetadata,
): void;
```

**Parâmetros:**

| Parâmetro | Tipo | Descrição |
|---|---|---|
| `content` | `string` | O conteúdo que foi lido ou buscado |
| `metadata.source` | `"web" \| "file" \| "channel" \| "skill" \| "memory"` | De onde o conteúdo veio |
| `metadata.path` | `string?` | Caminho do arquivo, nome da skill ou identificador do canal |
| `metadata.url` | `string?` | URL (para fontes web) |

**Mapeamento de fonte para taint:**

| Fonte | Taint Level |
|---|---|
| `web` | WEB |
| `file` | UNTRUSTED |
| `channel` | UNTRUSTED |
| `skill` | SKILL |
| `memory` | MEMORY |

### scanEgress()

Escaneia conteúdo de saída em busca de segredos vazados, PII e dados sensíveis antes do envio.

```typescript
const result = sl.scanEgress(content: string): EgressScanResult;
```

**Retorna:** `EgressScanResult` com `clean: boolean` e `findings: EgressFinding[]`.

### getSessionTaint()

Obtém o taint level efetivo da sessão atual.

```typescript
const taint = sl.getSessionTaint(): string;
```

**Retorna:** Um dos valores `"owner"`, `"trusted"`, `"untrusted"`, `"web"`, `"skill"`, `"memory"`.

### on()

Inscreve-se em eventos de segurança do event bus interno.

```typescript
const unsubscribe = sl.on(type: string, handler: (event) => void): () => void;
```

**Parâmetros:**

| Parâmetro | Tipo | Descrição |
|---|---|---|
| `type` | `string` | Tipo de evento para escutar (ex.: `"action.evaluated"`, `"taint.elevated"`, `"approval.requested"`) |
| `handler` | `function` | Callback invocado quando o evento dispara |

**Retorna:** Uma função de cancelamento de inscrição. Chame-a para parar de receber eventos.

### destroy()

Limpa recursos. Resolve todas as solicitações de aprovação pendentes como expiradas e limpa todos os handlers de eventos.

```typescript
sl.destroy(): void;
```

Sempre chame `destroy()` quando terminar de usar a instância SecurityLayer.

## withSecurityLayer()

Wrapper middleware que adiciona verificações de segurança a qualquer função. Intercepta chamadas de forma transparente — a função encapsulada tem a mesma assinatura que a original.

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

const protectedExec = withSecurityLayer(sl, originalExecFn, "exec");
const protectedRead = withSecurityLayer(sl, originalReadFn, "file.read");
```

**Assinatura:**

```typescript
function withSecurityLayer<TArgs extends unknown[], TReturn>(
  sl: SecurityLayer,
  fn: (...args: TArgs) => TReturn | Promise<TReturn>,
  tool: string,
  options?: {
    extractParams?: (...args: unknown[]) => Record<string, unknown>;
  },
): (...args: TArgs) => Promise<TReturn>;
```

**Comportamento:**
- `ALLOW` → chama a função original e retorna seu resultado
- `DENY` → lança `SecurityLayerError` sem chamar a função
- `REQUIRE_APPROVAL` → aguarda aprovação, lança exceção se negado

**Extração de parâmetros padrão:** Se o primeiro argumento for uma string, é mapeado para `{ command: arg }`. Se for um objeto, é usado diretamente como params.

## Classes de erro

Todos os erros do SDK estendem `SecurityLayerError`, que estende `Error`.

```typescript
import {
  SecurityLayerError,
  ConfigError,
  InitializationError,
  CheckError,
  ApprovalTimeoutError,
} from "@securitylayerai/sdk";
```

| Erro | Propriedades | Lançado quando |
|---|---|---|
| `SecurityLayerError` | `message` | Classe base; também lançado por `withSecurityLayer()` em DENY |
| `ConfigError` | `configPath?: string` | Arquivos de configuração não podem ser carregados ou parseados |
| `InitializationError` | `cause?: unknown` | Security pipeline falha na inicialização |
| `CheckError` | `tool?: string` | Avaliação de ação falha inesperadamente |
| `ApprovalTimeoutError` | `approvalId: string` | Solicitação de aprovação expira |

## Tipos

### CheckResult

```typescript
interface CheckResult {
  decision: "ALLOW" | "DENY" | "REQUIRE_APPROVAL";
  reason: string;
  approvalId?: string;
  layers: LayerResults;
  degraded: boolean;
  timing: {
    total: number;
    capability?: number;
    rules?: number;
    llm?: number;
  };
}
```

### LayerResults

```typescript
interface LayerResults {
  capability: {
    allowed: boolean;
    reason?: string;
  };
  rules?: {
    decision: string;
    reason?: string;
  };
  llm?: {
    decision: string;
    confidence: number;
    reasoning?: string;
  };
}
```

### EgressScanResult

```typescript
interface EgressScanResult {
  clean: boolean;
  findings: EgressFinding[];
}

interface EgressFinding {
  type:
    | "api_key"
    | "pem_block"
    | "private_key"
    | "password"
    | "high_entropy"
    | "pii"
    | "credential_pattern";
  confidence: number;
  location: { start: number; end: number };
  redacted: string;
}
```

### ExecutionContext

```typescript
interface ExecutionContext {
  sessionId?: string;
  skillId?: string;
  channelId?: string;
}
```

### ContentMetadata

```typescript
interface ContentMetadata {
  source: "web" | "file" | "channel" | "skill" | "memory";
  path?: string;
  url?: string;
}
```

### SecurityLayerOptions

```typescript
interface SecurityLayerOptions {
  configDir?: string;
  sessionId?: string;
  config?: LoadedConfig;
  eventBus?: EventBus;
}
```

### ApprovalOptions

```typescript
interface ApprovalOptions {
  timeout?: number; // milliseconds, default 300000 (5 min)
}
```
