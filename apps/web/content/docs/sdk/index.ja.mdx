## SDK を使う理由

CLI 統合（フック、シェルシム）はコード変更なしで機能しますが、境界レベルでの動作です。SDK は完全なコンテキストを持つ**インプロセスセキュリティ**を提供します:

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| exec のインターセプト | Yes | Yes | Yes |
| ファイル書き込みのインターセプト | No | Yes | Yes |
| Web フェッチのインターセプト | No | Yes | Yes |
| 完全なセッションコンテキスト | No | 部分的 | **Yes** |
| Taint tracking 精度 | 粗い | 中程度 | **操作単位** |
| バイパスリスク | PATH 操作 | なし | **なし（インプロセス）** |
| 任意のフレームワークで動作 | Yes（シェルのみ） | Claude Code のみ | **任意のフレームワーク** |
| 統合の手間 | ゼロ | コマンド 1 つ | コード変更 |

SDK は、エージェントアプリケーションを構築していて構造的なセキュリティ保証が必要な場合 — コマンドラインインターセプションだけでなく — に最適な選択肢です。

## クイック例

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

// Check an action before executing it
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`Blocked: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId as string);
  if (!approved) console.error("Approval denied");
}

// Clean up when done
sl.destroy();
```
