El SDK soporta tres patrones de integracion. Elige segun como tu framework de agentes maneja la ejecucion de herramientas.

## Verificacion directa

Llama a `sl.check()` antes de cada ejecucion de herramienta. Ideal para frameworks donde controlas el bucle de despacho de herramientas.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

async function executeTool(tool: string, params: Record<string, unknown>) {
  const result = await sl.check(tool, params);

  if (result.decision === "DENY") {
    throw new ToolBlockedError(result.reason);
  }

  if (result.decision === "REQUIRE_APPROVAL") {
    const approved = await sl.waitForApproval(result.approvalId as string);
    if (!approved) throw new ToolBlockedError("Approval denied");
  }

  return actualToolExecution(tool, params);
}
```

**Cuando usarlo:** Cuando controlas el bucle de despacho de herramientas y quieres control explicito sobre las decisiones de seguridad.

## Middleware wrapper

Envuelve ejecutores de herramientas existentes con `withSecurityLayer()`. El wrapper intercepta las llamadas de forma transparente -- sin cambios en el codigo que las invoca.

```typescript
import { createSecurityLayer, withSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

// Wrap any tool executor
const protectedExec = withSecurityLayer(sl, originalExecFunction, "exec");
const protectedWrite = withSecurityLayer(sl, originalWriteFunction, "file.write");

// Usage — identical API, security checks happen transparently
await protectedExec("git push origin main");
await protectedWrite("/tmp/output.txt", content);
```

Puedes proporcionar extraccion de parametros personalizada cuando la opcion por defecto no se ajusta:

```typescript
const protectedFetch = withSecurityLayer(sl, fetchFn, "web_fetch", {
  extractParams: (url: unknown) => ({ url }),
});
```

**Cuando usarlo:** Cuando quieres agregar seguridad a un codebase existente con cambios minimos. Envuelve los ejecutores de herramientas una vez y el resto del codigo permanece intacto.

## Basado en eventos

Conecta a los eventos del ciclo de vida de tu framework. Ideal para frameworks que emiten eventos `beforeToolUse` / `afterToolUse`.

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

// Pre-execution: check and potentially block
agentFramework.on("beforeToolUse", async (event) => {
  const result = await sl.check(event.tool, event.params);
  if (result.decision !== "ALLOW") {
    event.preventDefault();
    event.setError(`Security Layer: ${result.reason}`);
  }
});

// Post-execution: update taint tracking
agentFramework.on("afterToolUse", (event) => {
  if (event.tool === "file.read" || event.tool === "web_fetch") {
    sl.ingestContent(event.output, {
      source: event.tool === "web_fetch" ? "web" : "file",
      path: event.params.path,
      url: event.params.url,
    });
  }
});
```

**Cuando usarlo:** Cuando tu framework ya tiene un sistema de plugins/eventos. El patron basado en eventos mantiene la logica de seguridad desacoplada de la ejecucion de herramientas.

## Comparacion de patrones

| Aspecto | Verificacion directa | Middleware | Basado en eventos |
|---|---|---|---|
| Cambios de codigo | Moderados | Minimos | Minimos |
| Control | Total | Transparente | Desacoplado |
| Taint tracking | Manual | Automatico | Manual |
| Ideal para | Frameworks personalizados | Codebases existentes | Arquitecturas de plugins |
| Flujo de aprobacion | Explicito | Integrado | Manual |

## Combinando patrones

Puedes mezclar patrones en la misma aplicacion:

```typescript
const sl = await createSecurityLayer();

// Middleware for standard tools
const protectedExec = withSecurityLayer(sl, exec, "exec");
const protectedWrite = withSecurityLayer(sl, writeFile, "file.write");

// Direct check for custom logic
async function handleDangerousOperation() {
  const result = await sl.check("exec", {
    command: "sudo systemctl restart nginx",
  });
  // Custom handling...
}

// Event-based for taint tracking
framework.on("afterToolUse", (event) => {
  sl.ingestContent(event.output, { source: "file", path: event.params.path });
});

// Always clean up
process.on("exit", () => sl.destroy());
```
