## Por qu usar el SDK?

Las integraciones del CLI (hooks, shell shim) funcionan sin cambios de cdigo -- pero operan en el borde. El SDK te da **seguridad en proceso** con contexto completo:

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| Intercepta exec | S | S | S |
| Intercepta escrituras de archivos | No | S | S |
| Intercepta web fetch | No | S | S |
| Contexto completo de sesin | No | Parcial | **S** |
| Precisin de seguimiento de taint | Gruesa | Moderada | **Por operacin** |
| Riesgo de bypass | Manipulacin de PATH | Ninguno | **Ninguno (en proceso)** |
| Funciona con cualquier framework | S (solo shell) | Solo Claude Code | **Cualquier framework** |
| Esfuerzo de integracin | Cero | Un comando | Cambios de cdigo |

El SDK es la opcin correcta cuando ests construyendo una aplicacin agentiva y quieres garantas de seguridad estructural -- no solo intercepcin en lnea de comandos.

## Ejemplo rpido

```typescript
import { SecurityLayer } from "@securitylayerai/sdk";

const sl = new SecurityLayer();

// Verificar una accin antes de ejecutarla
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`Bloqueado: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId);
  if (!approved) console.error("Aprobacin denegada");
}
```
