## Por que usar el SDK?

Las integraciones del CLI (hooks, shell shim) funcionan sin cambios de codigo -- pero operan en el borde. El SDK te da **seguridad en proceso** con contexto completo:

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| Intercepta exec | Si | Si | Si |
| Intercepta escrituras de archivos | No | Si | Si |
| Intercepta web fetch | No | Si | Si |
| Contexto completo de sesion | No | Parcial | **Si** |
| Precision de taint tracking | Gruesa | Moderada | **Por operacion** |
| Riesgo de bypass | Manipulacion de PATH | Ninguno | **Ninguno (en proceso)** |
| Funciona con cualquier framework | Si (solo shell) | Solo Claude Code | **Cualquier framework** |
| Esfuerzo de integracion | Cero | Un comando | Cambios de codigo |

El SDK es la opcion correcta cuando estas construyendo una aplicacion agentiva y quieres garantias de seguridad estructural -- no solo intercepcion en linea de comandos.

## Ejemplo rapido

```typescript
import { createSecurityLayer } from "@securitylayerai/sdk";

const sl = await createSecurityLayer();

// Check an action before executing it
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`Blocked: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId as string);
  if (!approved) console.error("Approval denied");
}

// Clean up when done
sl.destroy();
```
