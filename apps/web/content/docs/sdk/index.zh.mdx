## 为什么使用 SDK？

CLI 集成（hooks、shell shim）无需代码修改即可工作 — 但它们在边界处运行。SDK 为您提供**进程内安全**，拥有完整的上下文：

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| 拦截执行 | 是 | 是 | 是 |
| 拦截文件写入 | 否 | 是 | 是 |
| 拦截网络请求 | 否 | 是 | 是 |
| 完整会话上下文 | 否 | 部分 | **是** |
| 污点追踪精度 | 粗粒度 | 中等 | **每次操作** |
| 绕过风险 | PATH 操纵 | 无 | **无（进程内）** |
| 兼容任何框架 | 是（仅 shell） | 仅 Claude Code | **任何框架** |
| 集成工作量 | 零 | 一条命令 | 代码修改 |

当您构建 AI 代理应用并需要结构性安全保障 — 而不仅仅是命令行拦截时，SDK 是正确的选择。

## 快速示例

```typescript
import { SecurityLayer } from "@securitylayerai/sdk";

const sl = new SecurityLayer();

// 执行操作前进行检查
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`已阻止: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId);
  if (!approved) console.error("审批被拒绝");
}
```
