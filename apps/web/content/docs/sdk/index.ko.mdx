## SDK를 사용해야 하는 이유

CLI 통합(hooks, shell shim)은 코드 변경 없이 작동하지만 경계에서만 동작합니다. SDK는 전체 컨텍스트와 함께 **프로세스 내 보안**을 제공합니다:

| | Shell Shim | Hooks | SDK |
|---|---|---|---|
| 실행 가로채기 | 예 | 예 | 예 |
| 파일 쓰기 가로채기 | 아니오 | 예 | 예 |
| 웹 요청 가로채기 | 아니오 | 예 | 예 |
| 전체 세션 컨텍스트 | 아니오 | 부분적 | **예** |
| Taint 추적 정밀도 | 대략적 | 보통 | **작업별** |
| 우회 위험 | PATH 조작 | 없음 | **없음 (프로세스 내)** |
| 모든 프레임워크 지원 | 예 (셸만) | Claude Code만 | **모든 프레임워크** |
| 통합 난이도 | 제로 | 명령 하나 | 코드 변경 |

SDK는 에이전트 애플리케이션을 구축하면서 구조적 보안 보장을 원할 때 올바른 선택입니다 — 단순한 명령줄 가로채기가 아닙니다.

## 빠른 예제

```typescript
import { SecurityLayer } from "@securitylayerai/sdk";

const sl = new SecurityLayer();

// 실행 전에 동작을 검사합니다
const result = await sl.check("exec", { command: "rm -rf /tmp/output" });

if (result.decision === "DENY") {
  console.error(`차단됨: ${result.reason}`);
} else if (result.decision === "REQUIRE_APPROVAL") {
  const approved = await sl.waitForApproval(result.approvalId);
  if (!approved) console.error("승인이 거부되었습니다");
}
```
